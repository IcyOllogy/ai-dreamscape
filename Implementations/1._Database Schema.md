# 1. Database Schema (Supabase)

This plan details the implementation of the relational schema and Row-Level Security (RLS) policies for AI Dreamscape, as specified in `IMPLEMENTATION_PLAN.md`.

## 🎯 Objectives
Implement a robust relational schema with RLS, setup basic authentication (signup/login), establish user roles, and create mockup dashboards for end-to-end testing of the auth and role flows.

## 🗄️ Database Tables

### `profiles` Table
Extends `auth.users` with application-specific data.
- `id`: uuid (PK, references auth.users)
- `username`: text (unique, for @slugs)
- `display_name`: text
- `avatar_url`: text
- `role`: enum ('free', 'bronze', 'silver', 'gold', 'admin')
- `tokens_balance`: int (default 0)
- `is_banned`: boolean (default false)
- `created_at`: timestamp

### `pricing_plans` Table
Defines available membership tiers.
- `id`: uuid (PK)
- `name`: text
- `price`: decimal
- `tokens_included`: int
- `features`: jsonb
- `is_featured`: boolean
- `discount_label`: text

### `token_transactions` Table
Ledger for tracking all token movements.
- `id`: uuid (PK)
- `user_id`: uuid (FK)
- `amount`: int (positive for purchase/replenish, negative for spend)
- `type`: enum ('purchase', 'replenishment', 'generation', 'refund')
- `description`: text

### `login_history` & `sessions` Tables
- `login_history`: Tracks `user_id`, `ip_address`, `user_agent`.
- `sessions`: Tracks active engagement time and duration.

### `admin_audit_logs` Table
Audit trail for administrative actions.
- `id`: uuid (PK)
- `admin_id`: uuid (FK)
- `action`: text
- `target_id`: text
- `metadata`: jsonb

## 🔐 Security (RLS Policies)
- Users can read/write their own `profiles` and `token_transactions`.
- Admins have full access to all tables.
- `pricing_plans` are publicly readable but only editable by Admins.

## 🚀 Implementation Sequence
1. **Supabase Setup**: Setup Supabase project and create all database tables and enums.
2. **Authentication Pages**: Create the frontend Signup and Login pages.
3. **Role Management**: Handle user roles in the application logic. We will manually create the initial 'admin' user directly in the Supabase dashboard once the tables and roles are set up.
4. **Mockup Dashboards**: Create mockup versions of the Admin Dashboard and User Dashboard to thoroughly test user signup, login flows, and role-based access restrictions.
