# Implementation Plan: 1.5 User Profile & Navigation UX

This phase focuses on enhancing the user experience by providing a dedicated profile management page and refining the main navigation with a modern profile dropdown.

## Acceptance Criteria
- [ ] **Profile Page**: Create `/profile` route accessible to all authenticated users.
- [ ] **Profile Form**: Allow users to view and update their `display_name` and `avatar_url`.
- [ ] **Navigation UX**: Replace the static "Dashboard" button in the navbar with a Profile Dropdown.
- [ ] **Avatar Support**: Display the user's avatar if available, or a sleek fallback/placeholder.
- [ ] **Dropdown Menu**: Include "Profile" (links to `/profile`) and "Log Out" (trigger existing logout logic).

## Proposed Changes

### 1. Navigation Refactor (`src/components/Navigation.tsx`)
- Implement a dropdown menu using a glassmorphism style.
- State: Fetch `profiles.avatar_url` to display in the navbar.
- Replace the "Dashboard" button with this new component.

### 2. Profile Page (`src/routes/profile.tsx`)
- Layout: Glassmorphic card design.
- Features:
  - Header: Large avatar preview.
  - Fields: Display Name, Username (Slug), Email (Read-only).
  - Actions: Save Changes button.

### 3. Logic & State
- Use `useAuth` hook to provide user data to the profile page and navbar.
- Implement an `updateProfile` function in `useAuth` or a separate utility.
