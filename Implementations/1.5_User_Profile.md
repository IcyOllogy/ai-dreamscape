# Implementation Plan: 1.5 User Profile & Navigation UX

This phase focuses on enhancing the user experience by providing a dedicated profile management page and refining the main navigation with a modern, high-fidelity profile dropdown.

## User Review Required
> [!IMPORTANT]
> **Avatar Strategy**: We have decided NOT to implement image uploads for now. Users will provide a direct URL for their profile picture.
> **Placement**: The profile dropdown will be implemented as a **top-right trigger** for maximum visibility and "standard" web UX.

## Acceptance Criteria
- [ ] **Profile Page**: Create `/profile` route accessible to all authenticated users.
- [ ] **Profile Form**: Allow users to view and update their `display_name` and `avatar_url`.
- [ ] **Current Stats**: Show user's **Current Tier** and **Token Balance** on the profile page.
- [ ] **Navigation UX**: Implement a Top-Right Profile Dropdown.
- [ ] **Avatar Support**: Display the user's avatar if available, or a sleek **Neon Placeholder** (Initial-based).
- [ ] **Dropdown Menu**: Include "Profile", "Dashboard", and "Log Out".

## Proposed Changes

### 1. Navigation Refactor (`src/components/site/Navigation.tsx`)
- Implement a `DropdownMenu` (Shadcn) in the top-right corner.
- **Trigger**: `Avatar` component with a neon glow effect.
- **Content**: 
  - Section header with Username/Role.
  - Links: `Profile` (`/profile`), `Dashboard` (`/dashboard`).
  - Separator.
  - Action: `Sign Out`.

### 2. Profile Page (`src/routes/profile.tsx`)
- **Layout**: Centered glassmorphic card.
- **Header Section**: Large stylized avatar preview.
- **Form Section**:
  - `Display Name`: Editable text input.
  - `Avatar URL`: Editable text input.
  - `Username`: Read-only slug.
  - `Email`: Read-only from auth metadata.
- **Action**: "Save Changes" with `Sonner` toast feedback.

- Implement real-time profile fetching to ensure the navbar stays in sync with changes made on the profile page.

### 4. Supabase Storage (New)
- **Bucket**: `avatars`.
- **Security**: 
  - `Authenticated`: Can upload/update/delete in `avatars/{auth.uid()}/`.
  - `User`: Can only `SELECT` (read) their own folder.
  - `Admin`: Can `SELECT` all folders.
- **Implementation**:
  - Update `useAuth.tsx` with `uploadAvatar` function.
  - Add `ImageUpload` component to `ProfileForm`.
